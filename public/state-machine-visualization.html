🎵 Radio Player State Machine

Why State Machines?

State machines provide a clear, predictable way to manage application state by making impossible states impossible and transitions explicit.

❌ Current: Multiple Booleans

isPlaying: false
isLoading: false
hasError: false

Problem: 2³ = 8 possible combinations, but only 5 are valid!

Can accidentally have isPlaying=true AND isLoading=true ❌

✅ State Machine: Single State

state: 'idle' | 'loading' |
'playing' | 'paused' | 'error'

Benefit: Exactly 5 possible values, all valid!

Can ONLY be in one state at a time ✅

States

IDLE
Initial state, no radio selected
LOADING
Connecting to stream, show spinner
PLAYING
Radio is playing, show pause button
PAUSED
Radio paused, auto-resume after 2s
ERROR
Failed to load, show error message

Transitions

IDLE → LOADING → PLAYING ⇄ PAUSED
LOADING → ERROR → LOADING

Events: PLAY, PAUSE, NEXT, PREV, SELECT_STATION, SUCCESS, FAILURE

Benefits

🚫 Impossible States Prevented

Can't be playing AND loading simultaneously

📍 Explicit Transitions

Clear rules for what states can transition to what

🎯 Centralized Logic

All state management in one place, not scattered

✅ Better Testing

Test each state and transition independently

🔒 Type Safety

TypeScript catches invalid states at compile time

📊 Visual Documentation

Diagram serves as living documentation

🐛 Easier Debugging

See current state in dev tools, time-travel debugging

🔄 No Race Conditions

State changes are atomic and predictable

Implementation Options

Option 1: Simple TypeScript

type State =
| { status: 'idle' }
| { status: 'loading'; stationIndex: number }
| { status: 'playing'; stationIndex: number }
| { status: 'error'; error: string };

const [state, setState] = useState<State>({ status: 'idle' });

Option 2: XState (Recommended)

const radioMachine = createMachine({
id: 'radio',
initial: 'idle',
states: {
idle: { on: { PLAY: 'loading' } },
loading: { on: { SUCCESS: 'playing', FAILURE: 'error' } },
playing: { on: { PAUSE: 'paused' } },
paused: { on: { PLAY: 'playing' } },
error: { on: { RETRY: 'loading' } }
}
});

const [state, send] = useMachine(radioMachine);
